Architecture Overview

Papercut is built on a modern technology stack designed for performance and reliability.

Technology Stack

  • Tauri — Lightweight desktop application framework using the system webview
  • React — Component-based user interface library
  • Vite — Fast build tool and development server
  • Pagefind — Static search library with build-time indexing
  • TypeScript — Type-safe JavaScript for maintainable code

Build Pipeline

The build process follows these steps:

  1. Vite compiles the React application and copies HTML documents to the dist directory
  2. Pagefind scans the HTML documents and generates a compressed search index
  3. Tauri bundles everything into a native desktop application

Search Index

The Pagefind search index is generated at build time. It consists of compressed index fragments that are loaded on demand during search operations. This approach keeps the initial application load time fast while supporting large document collections. The WASM-based search engine runs entirely in the browser context with no server required.

Desktop Integration

Tauri provides native desktop integration through the system webview (WebKit on Linux, WebView2 on Windows, WKWebView on macOS). This results in application binaries that are typically under 10MB, compared to Electron-based applications that can exceed 150MB.
