# Papercut v1.4.2 - Release Notes

- **Target release:** 1.4.2
- **Release branch:** `main`
- **Primary change:** Fresh iOS/TestFlight release version after the `v1.4.1` tag was already created during release workflow testing.

Papercut v1.4.2 carries the same intended app behavior as the v1.4.1 release patch and v1.4.0 native iOS TTS work. This version exists to publish the current fixed release workflow from a clean new tag instead of rewriting the already-created `v1.4.1` tag.

## Highlights

- **Keeps iOS release workflow hardening.** iOS PR and release jobs use `macos-26`, enforce iOS SDK 26 or newer, and fail early if expected Swift compatibility libraries are missing.
- **Keeps release-class iPhoneOS PR validation.** PR CI builds the arm64 simulator target and an unsigned generic iPhoneOS device target with native TTS before release.
- **Keeps native TTS device linking fixes.** The PR-only device check prebuilds the Rust static library with Cargo using the configured iOS minimum deployment target, stages Cargo's `libapp_lib.a` output as `libapp.a`, and then runs direct `xcodebuild build`.
- **Keeps signed release path unchanged.** The protected release workflow still performs the signed App Store Connect export/upload path with the `apple-release` environment secrets.

## Compatibility

No data migration is expected. Models remain downloaded on demand and verified before install. iOS TestFlight validation is still required for real-device model download, audiobook generation, playback, highlighting, and background controls.

## Verification

- PR iOS CI passed after the iOS device build hardening.
- v1.4.2 is intended to be released from latest `main` with a fresh tag, avoiding the stale `v1.4.1` tag created during workflow testing.
