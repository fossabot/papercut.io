# Papercut v1.3.1 - Release Notes

- **Target release:** 1.3.1
- **Baseline:** [`v1.3.0`](https://github.com/muhannadnouri/papercut.io/releases/tag/v1.3.0)
- **Release branch:** `branch-release-v1.3.1`
- **Primary change:** macOS patch release for native TTS dylib packaging.

Papercut v1.3.1 is a patch release for the first signed/notarized macOS distribution path. App behavior is otherwise intended to match v1.3.0.

## Highlights

- **Fixes macOS launch failure from missing dylibs.** The desktop build now pre-stages macOS `libsherpa-onnx-c-api.dylib` and `libonnxruntime.dylib` before Tauri scans bundle resources, so the installed app can resolve them from `Papercut.app/Contents/Resources` via the existing `@loader_path/../Resources` rpath.
- **Adds CI guards for macOS runtime libraries.** PR CI and release CI now fail if required macOS dylibs are missing from the built `.app`; release CI also verifies signatures on those dylibs before DMG notarization/upload.
- **Keeps signed/notarized DMG flow.** Developer ID signing, app notarization, DMG notarization, stapling, and Gatekeeper checks remain part of the protected `apple-release` release job.

## v1.3.0 Note

The v1.3.0 macOS DMGs could pass signing/notarization checks while still failing at launch because required native TTS dylibs were missing from the app bundle. Prefer v1.3.1 for macOS. Linux, Windows, and Android app behavior is unchanged by this patch.

## Compatibility

No saved-data or audiobook cache migration is expected. The native audiobook cache version remains unchanged.

## Verification

- GitHub CI builds macOS Intel and Apple Silicon DMGs.
- CI verifies `libsherpa-onnx-c-api.dylib` and `libonnxruntime.dylib` exist in `Papercut.app/Contents/Resources`.
- Release CI verifies the app bundle, required dylibs, notarization, stapling, and Gatekeeper assessment before uploading DMGs.
- MacInCloud/manual smoke test should launch the Apple Silicon DMG after dragging `Papercut.app` out of the mounted image.
