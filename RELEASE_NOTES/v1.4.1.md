# Papercut v1.4.1 - Release Notes

- **Target release:** 1.4.1
- **Release branch:** `feature/ios-native-tts`
- **Primary change:** iOS App Store upload compatibility with Xcode/iOS SDK 26.

Papercut v1.4.1 is a release workflow patch for the first iOS/TestFlight build. App behavior is intended to match v1.4.0.

## Highlights

- **Builds iOS on macOS 26 runners.** iOS PR and release jobs now use GitHub's `macos-26` runner so App Store uploads are built with Xcode/iOS SDK 26 or newer.
- **Fails early on stale iOS SDKs.** iOS jobs print the selected Xcode version and reject SDKs older than iOS 26 before building/uploading.
- **Keeps v1.4.0 native TTS work intact.** iOS still builds with native sherpa-onnx TTS, thin Cargo link archives, retry-safe TestFlight build numbers, and target-aware Sherpa slice selection.

## Compatibility

No data migration is expected. Models remain downloaded on demand and verified before install. iOS TestFlight validation is still required for real-device model download, audiobook generation, playback, highlighting, and background controls.

## Verification

- PR iOS CI passed before this release patch.
- Release upload failed only because the previous runner used Xcode 16.4 / iOS SDK 18.5.
- The workflow now guards against that runner/toolchain mismatch before build/upload.
