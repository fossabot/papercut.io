# Papercut v1.3.2 - Release Notes

- **Target release:** 1.3.2
- **Baseline:** [`v1.3.1`](https://github.com/muhannadnouri/papercut.io/releases/tag/v1.3.1)
- **Release branch:** `branch-release-v1.3.2`
- **Primary change:** sign bundled macOS native TTS dylibs before notarization.

Papercut v1.3.2 is a macOS packaging patch release. App behavior is otherwise intended to match v1.3.1.

## Highlights

- **Fixes macOS notarization failure for bundled dylibs.** Release builds now bundle every `.dylib` from the sherpa macOS archive, including versioned ONNX Runtime siblings such as `libonnxruntime.1.24.4.dylib`, then Developer-ID-sign staged native TTS dylibs with hardened runtime and secure timestamps before Tauri bundles and notarizes the `.app`.
- **Keeps macOS dylib inclusion guards.** CI and release jobs fail if `libsherpa-onnx-c-api.dylib`, `libonnxruntime.dylib`, or the versioned `libonnxruntime.*.dylib` dependency is missing from `Papercut.app/Contents/Resources`.
- **Strengthens release verification.** The protected release job walks `otool -L` dependencies for the app and every bundled dylib, then verifies bundled dylib signatures before DMG notarization/upload.

## v1.3.1 Note

The v1.3.1 macOS release path fixed missing dylib staging but exposed a second notarization requirement: Apple requires bundled `.dylib` files to be signed with a valid Developer ID certificate and secure timestamp. Prefer v1.3.3 or newer for macOS installs.

## Compatibility

No saved-data or audiobook cache migration is expected. The native audiobook cache version remains unchanged.

## Verification

- CI builds macOS Intel and Apple Silicon DMGs.
- CI verifies required macOS dylibs and the versioned ONNX Runtime dylib exist in `Papercut.app/Contents/Resources`.
- Release CI signs bundled dylibs, verifies transitive `otool -L` dependencies, and verifies dylib signatures before app notarization.
- Release CI notarizes/staples the app and DMG, then runs Gatekeeper checks before upload.
