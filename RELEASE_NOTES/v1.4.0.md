# Papercut v1.4.0 - Release Notes

- **Target release:** 1.4.0
- **Baseline:** [`v1.3.3`](https://github.com/muhannadnouri/papercut.io/releases/tag/v1.3.3)
- **Release branch:** `feature/ios-native-tts`
- **Primary change:** iOS TestFlight builds with native sherpa-onnx TTS.

Papercut v1.4.0 makes iOS a signed/TestFlight target with the same offline document import, search, saved audiobook, and native TTS direction used by desktop and Android.

## Highlights

- **Adds iOS native TTS build support.** The iOS build downloads the official sherpa-onnx iOS static XCFramework archive, verifies its SHA-256, prepares Cargo-compatible static-library aliases, and builds the IPA with `native-tts-static`.
- **Tests iOS native TTS in PR CI.** The macOS iOS CI job now compiles the unsigned simulator target with the native TTS static-library slice, catching Rust/Tauri/Xcode link issues before release.
- **Keeps TestFlight build numbers flexible.** The release workflow patches iOS `CFBundleVersion` from the GitHub run number, so multiple TestFlight uploads can share the same user-facing `1.4.0` marketing version.
- **Uses conservative iOS synthesis defaults.** iOS native TTS defaults to one ONNX Runtime thread, matching Android's mobile-safe setting for long audiobook generation.

## Compatibility

No saved-data or audiobook cache migration is expected. Models are still downloaded on demand and verified before install. iOS does not package voice models inside the IPA.

## Verification

- PR CI builds the unsigned iOS simulator app with native TTS enabled.
- Release CI builds and signs the App Store Connect IPA with native TTS enabled.
- Real-device TestFlight validation is required before App Review: import, search, model download, audiobook generation, playback/background controls, and highlighting.
